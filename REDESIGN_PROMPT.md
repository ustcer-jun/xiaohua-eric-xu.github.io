## 二、核心问题诊断

### 2.1 当前网站的主要问题（老师反馈 + 自身分析）

1. **过于商业化**：当前设计使用了大量企业级 UI 元素（全屏 Hero Banner、CTA 按钮、渐变遮罩、卡片悬停动画），看起来像一个科技公司或 SaaS 产品官网，而非学术课题组主页
2. **信息密度不足**：首页大量留白和装饰性元素，实际学术内容展示不够
3. **缺乏学术气质**：缺少学术网站典型的内容优先布局、论文/研究展示、简洁的信息架构
4. **过度设计**：过多的动画效果、渐变、阴影让页面显得花哨，与学术研究的严肃气质不符

### 2.2 参考课题组主页分析总结

典型学术课题组特征：
**✅ 应该借鉴的优点：**
1. **内容优先的布局**：页面以文字内容为主，图片作为辅助说明，而非装饰
2. **简洁的导航结构**：顶部一行文字导航，无下拉菜单，无汉堡菜单，简洁明了
3. **信息密集但有序**：页面内容充实，通过清晰的分区和标题组织
4. **学术配色**：深色（深蓝/深青）头部 + 浅色内容区，专业沉稳
5. **双语支持**：英文为主，中文翻译作为辅助
6. **科研可视化**：使用科研相关的图片（地震图、InSAR 形变图、地质剖面图）而非装饰性图片
7. **论文/成果展示突出**：出版物列表详细，包含完整的引用信息和链接
8. **团队成员展示**：照片 + 姓名 + 职位 + 研究方向 + 联系方式的标准化格式
9. **新闻/活动记录**：以时间线形式记录组内活动，配以真实照片
10. **合作单位展示**：以 Logo 墙形式展示合作企业和机构

**❌ 不应照搬的缺点：**
1. 固定宽度布局（不响应式）— 我们应保持响应式设计
2. 暖色调（桃色/米色）背景 — 我们使用更现代的浅灰/白色
3. 过于密集的文字排列 — 适当增加行距和间距
4. 缺乏视觉层次 — 我们应保持一定的视觉引导

## 三、设计规范

### 3.1 整体设计理念

**关键词：学术、专业、简洁、内容驱动**

- 以内容为核心，减少装饰性元素
- 保持学术网站的庄重感和专业感
- 信息架构清晰，便于浏览和查找
- 适当使用科研可视化图片增强学术氛围
- 保持响应式设计以适应不同设备

### 3.2 配色方案

**主色调（替换当前的商业化蓝红配色）：**

| 角色 | 色值 | 使用场景 |
|------|------|----------|
| **主色（深青蓝）** | `#1B4F72` | 导航栏背景、页脚背景、标题文字 |
| **辅助深色** | `#154360` | 导航悬停态、页脚渐变 |
| **强调色（学术红）** | `#922B21` | 活跃导航项、重要标题装饰线 |
| **内容区背景** | `#F8F9FA` | 页面主背景（极浅灰） |
| **白色背景** | `#FFFFFF` | 卡片背景、内容区块 |
| **正文文字** | `#2C3E50` | 主要文字内容 |
| **辅助文字** | `#5D6D7E` | 日期、描述、次要信息 |
| **分割线** | `#D5D8DC` | 内容区块之间的分割线 |
| **链接色** | `#2874A6` | 超链接、可点击文字 |

**设计原则：**
- 不使用渐变背景（除页脚外）
- 不使用大面积彩色背景
- 不使用霓虹色或高饱和度颜色
- 整体色调偏冷、偏深，传达学术严谨感

### 3.3 字体规范
```
/* 英文字体 */
font-family: "Source Sans Pro", "Helvetica Neue", Arial, sans-serif;

/* 中文字体 */
font-family: "Noto Sans SC", "Microsoft YaHei", "PingFang SC", sans-serif;

/* 组合字体栈 */
font-family: "Source Sans Pro", "Noto Sans SC", "Helvetica Neue", Arial, sans-serif;
```

**字号层级：**
| 元素 | 大小 | 字重 | 备注 |
|------|------|------|------|
| 网站标题 | 24px | 700 | 导航栏左侧 |
| 页面标题 | 28px | 700 | 各页面顶部 |
| 区块标题 | 20px | 600 | 内容区块标题 |
| 小节标题 | 16px | 600 | 卡片标题、列表标题 |
| 正文 | 15px | 400 | 主要内容 |
| 辅助文字 | 13px | 400 | 日期、标签、描述 |

### 3.4 布局规范

| 参数 | 值 |
|------|------|
| 最大内容宽度 | `max-w-6xl`（1152px） |
| 内容区左右内边距 | `px-6`（24px） |
| 区块上下间距 | `py-12`（48px） |
| 卡片圆角 | `rounded-lg`（8px） |
| 卡片阴影 | `shadow-sm`（极淡阴影）或无阴影 |
| 分割线样式 | `border-b border-gray-200` |

### 3.5 动画规范（大幅减少）

**保留的动画：**
- 页面切换时的淡入效果（Framer Motion，`opacity: 0→1, duration: 0.3s`）

**移除的动画：**
- ❌ 卡片悬停上浮效果（`whileHover={{ y: -8 }}`）
- ❌ Hero 区域的复杂入场动画
- ❌ 滚动触发的元素动画（`whileInView`）
- ❌ 任何弹性/弹性动画效果

**替代方案：**
- 仅使用 CSS `transition` 实现简单的悬停颜色变化
- 导航链接悬停时仅改变文字颜色或添加下划线

## 五、关键设计对比（当前 vs 目标）

| 维度 | 当前设计（商业化） | 目标设计（学术化） |
|------|-------------------|-------------------|
| **Hero 区域** | 全屏 70vh + 渐变遮罩 + CTA 按钮 | 无 Hero，直接展示课题组简介 |
| **配色** | 蓝色 + 红色渐变，高对比度 | 深青蓝 + 学术红，沉稳专业 |

| **动画** | Framer Motion 大量动画 | 仅保留页面淡入，CSS transition |

| **图片使用** | 装饰性大图 | 科研可视化图片 + 组内照片 |
| **论文展示** | 简单列表 | 标准学术引用格式 |
| **页脚** | 多列链接 + 社交图标 | 简洁联系信息 + 版权 |

## 十、重要提醒

1. **不要引入新的第三方库**，仅使用现有依赖（React, Tailwind, Framer Motion, Lucide React, React Router）
2. **保留现有的数据文件结构**（JSON 格式），确保数据兼容

6. **保留 GitHub Pages 部署配置**，确保构建后能正常部署
7. **图片使用现有资源**，不要引入外部图片（除 Google Fonts 外）
